Code — Vulnerable vs Secure

🔴 VULNERABLE — /api/search
app.get('/api/search', (req, res) => { const q = req.query.q || ''; // ❌ Raw user input injected into HTML res.send(` <p>Results for: ${q}</p> `); });
⚠ ${q} is raw. <script> in it becomes live HTML.
🟢 SECURE — /api/search/safe
app.get('/api/search/safe', (req, res) => { const q = req.query.q || ''; // ✅ Escape before inserting into HTML const safe = escapeHtml(q); res.send(` <p>Results for: ${safe}</p> `); });
✓ < → &lt; — browser shows text, never executes.

Attack Flow

  • 1
    Attacker crafts URL: /api/search?q=<script>alert(document.cookie)</script>
  • 2
    Sends link to victim via phishing email, social media, or QR code.
  • 3
    Victim clicks — browser sends the request to the trusted server.
  • 4
    Server returns HTML with the <script> embedded verbatim.
  • 5
    Browser executes the script in the trusted domain's context — cookie theft complete.
