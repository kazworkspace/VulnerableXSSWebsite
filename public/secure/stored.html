Critical Concept — Encode at Output, Not at Input

💡 Why store raw and encode at render?
// ❌ WRONG: sanitize before storing // Problem: destroys original data + double-encoding bugs db.save({ body: escapeHtml(userInput) }); // "I <3 coding" stored as "I &lt;3 coding" // Shown to user as: "I &lt;3 coding" ← wrong // ✅ CORRECT: store raw, escape at render time db.save({ body: userInput }); // stored: "I <3 coding" html += escapeHtml(row.body); // rendered: "I &lt;3 coding" // Shown to user as: "I <3 coding" ← correct

Code — The Fix

🔴 Before (Vulnerable)
// ❌ Raw injection into HTML `<div class="comment-body"> ${c.body} </div>`
🟢 After (Secure)
// ✅ Escape every field at render `<div class="comment-body"> ${escapeHtml(c.body)} </div>`

When You Must Allow HTML (Rich Text)

📜 Use DOMPurify — never regex
// ✅ Allowlist-based HTML sanitizer import DOMPurify from 'dompurify'; const clean = DOMPurify.sanitize(userHtml, { ALLOWED_TAGS: ['b', 'i', 'em', 'strong', 'a'], ALLOWED_ATTR: ['href'] }); // ❌ Never use regex to strip tags — bypassable input.replace(/<script.*?>.*?<\/script>/gi, '') // Bypass: <scr<script>ipt>alert(1)</scr</script>ipt>
ⓘ
Regex-based stripping is always bypassable. Use a proper HTML parser like DOMPurify for rich text.
