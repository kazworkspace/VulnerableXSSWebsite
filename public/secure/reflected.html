The Fix

🔴 Before (Vulnerable)
const q = req.query.q || ''; // ❌ Raw — tags execute res.send(`<p>${q}</p>`);
🟢 After (Secure)
const q = req.query.q || ''; const safe = escapeHtml(q); // ✅ res.send(`<p>${safe}</p>`);

The escapeHtml Function

📜 Encode all 5 HTML special characters
function escapeHtml(str) { return String(str) .replace(/&/g, '&amp;') // ← MUST be first .replace(/</g, '&lt;') .replace(/>/g, '&gt;') .replace(/"/g, '&quot;') .replace(/'/g, '&#x27;'); }
ⓘ
& must be replaced first — otherwise later replacements would double-encode it (e.g., &lt; → &amp;lt;).

Defense in Depth

📜 Additional headers on secure routes
// Blocks inline scripts if escaping is bypassed res.set('Content-Security-Policy', "script-src 'self'; object-src 'none'"); // Session cookie inaccessible from JS res.cookie('session', token, { httpOnly: true, // ← JS cannot read this secure: true, sameSite: 'strict' });
