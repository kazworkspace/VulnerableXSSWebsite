Rule: Match the defense to the context. HTML body → escapeHtml(). Attribute → quote + escapeHtml(). URL/href → scheme allowlist first, then escapeHtml().

Case A — Unquoted Attribute → Fixed

🟢 Fixed: Quote + escapeHtml()

Fix: always quote attributes + escapeHtml() on the value.
Quotes trap the value — a space can't start a new attribute. " → &quot; — cannot close the quote.

Click to load — payload renders as inert text

✓ Event handler never attaches — payload displays as literal text in the style value.

Case B — Quoted Attribute Breakout → Fixed

🟢 Fixed: escapeHtml() encodes "

Fix: escapeHtml() converts " → &quot;.
The payload's quote is trapped inside the value string — no new attribute can be injected.

Click to load — " becomes &quot; in the rendered HTML

✓ " becomes &quot; — attribute never breaks, no new attributes injected.

Case C — href javascript: Injection → Fixed

🟢 Fixed: URL allowlist + escapeHtml()

Fix: safeUrl() checks scheme first — only http:/https: pass. Then escapeHtml() encodes any remaining HTML chars. Order matters — allowlist first, then encode.

Click to load — all dangerous schemes are blocked

✓ All blocked schemes → href becomes #blocked — click is harmless.

Case D — Unquoted data-* Attribute → Fixed

🟢 Fixed: Quote + escapeHtml()

Fix: wrap the attribute value in quotes and apply escapeHtml().
data-toolbar="${escapeHtml(toolbar)}"
Quotes trap the value — a space can't start a new attribute. Any injected " becomes &quot; and cannot close the quote.

Click to load — payloads are trapped as inert text

✓ Space stays inside the quoted value — no new attribute injected.

🟢 After (Secure)

The safeUrl() Function

📜 Allowlist approach — reject anything not http/https
function safeUrl(url) { try { const parsed = new URL(url); // URL spec normalizes scheme to lowercase: // "JAVASCRIPT:" → "javascript:" → blocked ✅ if (['http:', 'https:'].includes(parsed.protocol)) return url; return '#blocked'; // javascript:, data:, vbscript: } catch { // Allow relative paths like /page — reject //evil.com if (url.startsWith('/') && !url.startsWith('//')) return url; return '#blocked'; } }
